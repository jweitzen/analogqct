1.   Introduction

FilterLog enables to edit the string filters used by AnaLogQCT. A string filter consists in a list of strings. Each string possesses the following attributes:

 

·      String: this is the string text itself that will be looked for in the log. Once the string is found in a given line in the log, the whole line will be extracted and placed in the HTML output.

·      Colour: the colour the string will appear in the HTML output.

·      Comment: this string will be appended in red colour at the end of the line containing the string.

·      Replace / With: this 2 fields work together. The string in the “Replace” field will be substituted with the string in the “With” string. The replace will be performed on the entire line extracted from the log.

1.   Adding a string

Double click the bottom line displaying “…”.

2.   Editing an item

Double click the item.

3.   Deleting lines

Select the lines to be deleted (hold down “Ctrl” for multiple line selection) and press “Del”.

4.   Important notes

·      Only checked strings will be used during the log analysis. This is useful in order to temporarily disable strings without having to delete them from the filter. In order to keep a string but disable it, just uncheck the box next to it.

·      During the string search you can use Perl regular expressions. In order to use a Perl special character escape it with ‘!’.

o     Example: if you write ‘*’ in the string, the character will be matched as it is. If you write ‘!*’ then it becomes a Perl special character taking the meaning of Perl’s “*” which is: “any character”.

·      Selecting View/Filter information a dialog box opens. Some basic information about the log can be entered here.

·      Only save filters with .flt extension!
