1.   Introduction

AnaLogQCT helps the analysis of QXDM/QCAT logs. AnaLogQCT generates easy-to-read HTML file containing key information extracted from the log. The user has total control over the information contained in the HTML output.

In the above example the user has chosen to only view information relative to Layer 3 messages, NAS messages plus the information relative to the Active Set (timestamp 12:56:004) and event information (timestamp 12:56:26.326).

 

AnaLogQCT provides the following features:

 

  • The user can select a colour to associate with each piece of information extracted from the log. In the example above the user has chosen to print all RRC HO events in yellow (see timestamp 12:56:26:326)..
  • The user can associate a comment to each piece of information extracted from the log. All comments are printed in red (see timestamp 12:56:26:004).
  • Perl Regular Expressions can be used when extracting string from the log. This feature allows the user to easily extract lines with common characteristic.

 

The timestamps in the HTML output are hyperlinks to the equivalent line in the full text log allowing detailed analysis of the log. When the user selects one of the hyperlinks, the text in the second frame will jump to the corresponding line.

 

In order to analyse logs AnaLogQCT utilises string filters. A string filter is a list of text strings. For each string, AnaLogQCT scans the entire log line by line. If the string is found in a given line, the entire line is extracted from the log and put in the HTML output file. The result is an HTML file containing all extracted lines in chronological order. Each line will have a hyperlink which will let the user jump to the same line in the full log for further analysis. The user can edit the string filters using the associated program, FilterLog  (see below in the Create/Edit a string Filter paragraph for more details on string filters).

1.   Required PC setup

The program has been tested with Windows XP professional.

 

The following programs are required to be installed on your machine in order to work:

 

  • Microsoft Internet Explorer.
  • Perl. Please note that AnaLogQCT has been tested with 5.8 of Perl. Earlier version might impair AnaLogQCT operation.
  • QCAT.or APEX

2.   First time using the program

Before you can start analyzing logs some basic parameters need to be set. Please do the following.

 

  • Select View/Settings.
  • Choose the QCAT/APEX version installed on the machine in use.

3.   Settings

Selecting View/Settings the Settings dialog opens. The following options can be set through the Settings dialog:

 

  • The QCAT or APEX version installed on the machine. If an incorrect QCAT or APEX version is selected the output window will be empty.
  • The frame organisation mode. Choose whether to tile the links and full-text windows vertically or horizontally.
  • The ratio between the links and full-text windows on screen. The dimension of the two windows can be selected. Notice that this setting only affects the initial visualization. The user can move the divisor after the output has been displayed.

4.   Analyse a log

  • Select File/Open and open a DLF log.
  • In the “Filtering options” dialog box browse to the folder containing the QCAT filters (to be done only once).
  • Select the QCAT packets you want to focus on.
  • Select the string filters you want to apply to the log.
  • Select whether to perform a “Case sensitive” search or not.
  • Press OK.

 

N.B: due to the large size of QXDM/QCAT logs, processing of logs can take a long time. Especially if “Extended debug messages” packets are selected.

 

N.B.: the information you see in the output HTML file is the combination of the QCAT and string filters you apply to the log. The user is required to be aware of which QCAT packets need to be considered for each string filter to be effective. As an example, in the below example the user is interested in Out Of Service (OOS) information along with Layer 3 and NAS messages. To extract the correct information the user needs to:

 

  • In the left hand side box of the “Filtering options” dialog box, select the QCAT packets that contain the Layer 3 messages (that is, the “WCDMA Signalling Messages” packets) and select the QXDM/QCATA packets that contain the OOS messages (that is, the “Extended Debug Messages” packets (“Message view” packets)).
  • In the right hand box of the “Filtering options” dialog box, select the string filters that will extract the Layer 3 messages and the OOS messages. Layer 3 messages can be extracted using the “Basic.flt” filter while for OOS messages the user will need to create a string filter containing the appropriate strings. As an example, below all lines containing the string “CR RANK: S fail rank” and “OOS” have been extracted providing information on when the UE has detected a failure of the S criteria.

5.   Create/edit a string filter

FilterLog, the AnaLogQCT companion program,  enables to easily edit the string filters allowing to:

 

  • Create string which use Perl Regular Expressions.
  • Associate a colour to each string.
  • Associate a comment to each string.
  • Replace a part of an extracted line using the “Replace” and “With” fields.

 

FilterLog  can be started directly from AnaLogQCT selecting Filters/Edit filters or using the toolbar button. Please refer to the FilterLog  manual for details on editing filters.

6.   Tips

  • The easiest way to run AnaLogQCT is by right-clicking a DLF log.
  • Keep all filters in the same folder. When analyzing a log you can only select the filters contained in a given folder. Selection of filters from multiple folders is not possible.
  • Try to avoid building too large filters. Spread strings across multiple filters according to their purpose.

7.   Troubleshooting AnaLogQCT

  • Check if you have the QCAT/APEX program installed and with a valid license. Simply try to open QCAT/APEX.
  • Select the right version of QCAT/APEX in the Settings dialog box (to open the Settings dialog box select View/Settings).
  • If the “Available string filters” box in the “Filtering options” box is empty, please select a folder containing string filters in the “Filtering options” dialog box.
  • Check that you have Perl installed on your machine. The program has been tested with Perl 5.8. Earlier versions might not work properly.
  • Reinstall AnaLogQCT. You might be missing some installation file.
